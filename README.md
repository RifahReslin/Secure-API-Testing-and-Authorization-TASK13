# Secure API Testing and Authorization Validation

This repository contains documentation for Task 13: Secure API Testing & Authorization Validation.
The task focuses on testing REST APIs for authentication issues, authorization flaws, and common security misconfigurations.

## Objective
- Understand how REST APIs work
- Test API authentication
- Validate authorization controls
- Identify broken authorization issues
- Check input validation
- Analyze HTTP response codes
- Map findings to OWASP API risks

## Tools Used
- Postman
- cURL
- Insomnia

## Task Overview
APIs were tested using different request methods and inputs to identify security weaknesses related to authentication and authorization.

## Key Learnings
- Importance of API security
- Identifying broken authorization issues
- Understanding rate limiting
- Mapping vulnerabilities to OWASP API risks

## Outcome
This task improved practical skills in API security testing and authorization validation.
